# elenchus-ai
## Introduction:
ElenchusAI will help people solve their personal problems with the Socratic Method, elenchus, where they come to their own conclusions after being asked a series of thought-provoking, introspective, and extrospective questions. A large problem for some people in everyday decision-making, for choices large and small, is simply not facing the true issue or the core reason for procrastinating. By guiding people toward their truer or more factually correct thoughts critiquing flaws in reasoning, this application will prevent people from reasoning themselves into harmful stasis while also not validating or negating users’ concerns. This application is geared toward adults.

## Application Context and Environmental Constraints:
This version of the application will be available to desktop computer users.
This application does not need to be accessible to multiple device types at this point. 
The application will use ollama and gemma 4, exclusively, for multimodal user input. The application cannot come with ollama and gemma 4 pre-installed; it must check to see if the model is already locally available on-device. If it is not locally available, this application must fetch and download ollama and gemma 4 libraries onto the device (not within the application) to be used by the application.

## Functional Requirements:
Upon opening the application, the user will be asked, “What do you need today?” The options provided to the user are as follows: “A Rock to vent and complain to”, “A Problem Solver”, “A Therapist Friend, not a therapist”, “Just a Buddy”. Each option will reroute the user to a different page with relevant titles: “Rock”, “Problem Solver”, “Therapist Friend”, “Buddy”. 
Each option will define a different set of rules for gemma 4 to follow when conversing with the user. The set of rules will be included for context at every user input.
There must be an input box that can take user input to give to gemma 4.
There must be local safeguards around what can be inputted to gemma 4. If the user displays armful behaviors, all application modes should drop their personality, let the user know that something isn’t right, let the user know what the error is, and provide relevant, human resources, such as the suicide hotline, a therapist, and other resources. The user may not use this application for homework, work, or coding. The application is purely conversational and designed to guide the user toward problem-solving or alleviating emotional stress by talking through their issues. The application must not condone input of lewd language or graphics. The user is allowed to input obscene words, but the AI should ignore obscenity and focus on the core issue the user is presenting.
The model should never include the thought process in the output. The model should only output to the application the response that it would give the user. This output will be directly given to the user.
The application will never save any data about the user beyond the user session. All chat history is only local to the current chat and will be deleted upon application termination.
All modes will be mildly conversational to deliver a more human-like experience. However responses should never exceed the length of two paragraphs.
The Rock should allow the user to vent. Upon every prompt, the Rock should be able to pinpoint core points of emotional stress, tension, or importance in the user’s input and ask questions pertaining to it, with the end goal for them to relax themselves by getting the problems off of their chest. Recursive questions for this application mode should focus on giving the user space and letting the user expand and talk about various points of their original and following prompts so that the user can completely cover the original issue and the side issues. After it seems like the user has said everything they’d like to say about their situations, emotions, and concerns, the user should be asked what they’re going to do about it. At this point, a button should be available: “I don’t know, you tell me”. However, the chat box should still be available. The user can either map out their own solution plan that the AI can look at critically to decide whether to affirm (“that sounds like a good idea”) and offer follow-up suggestions or supportively disagree and offer other positive points of direction that the user would seem likely to take. The model should never explicitly say when the user is wrong or right.
The Problem Solver should allow the user to present their issue and guide the user to come up with a viable solution to their own problem by asking a series of introspective and extrospective questions. Nothing the user says should be taken as the truth; the AI should critically examine all of the user’s points given all the context the user has given throughout the chat. The AI should ask questions about what the user thinks and pay critical attention to each of the user’s reasons, context, opinions, facts, and personal concerns. If a user comes up with a faulty conclusion, where it may be negative, destructive, or simply manifest as a static coping mechanism, the AI will attempt to reroute the user’s thought process and have them think again. If a user comes up with a somewhat positive conclusion that is likely to take them in a better direction, then the Ai may affirm and offer 2-4 suggestions forward.
The Therapist Friend should focus on being emotionally responsive and attentive toward the user throughout the interaction, focusing on how the user feels and what can be done once the entirety of the issue is covered. Essentially, it fuses the functionality of the Rock and the Problem Solver to form a gentler, more nurturing personality.
The Buddy should simply be conversational and entertain conversations. However, if at any point, the user seems like they would better benefit from speaking to the Rock, the Problem Solver, or the Therapist Friend, it should ask whether the user would like to switch modes and offer a small menu. If the user declines, conversation may continue. If the user accepts and chooses to switch, then the user should be notified that the application mode has changed, and gemma 4’s rules should be overwritten with the set of rules from the chosen personality. The user’s input history should not be erased.



## Software Qualities and Non-Functional Requirements:
The UI needs to be simple and clean, with a warm, calming color palette to appeal to the target audience (emotionally charged users).

## Assumptions and Risks:
This application assumes that the user has common sense and does not use this application as a therapist or replacement for professional treatment.

## Future Directions and Expected Changes:
Further prompt engineering must be conducted on the personas that gemma 4 creates and adopts through chats to refine the user experience.
The UI must be more visually responsive to the user, especially when the AI model is thinking.
